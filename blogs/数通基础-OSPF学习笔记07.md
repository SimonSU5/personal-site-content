在网络规模扩大的情况下，LSA和LSDB规模变大。因此需要在不影响IP可达性的情况下，减少LSA的数量。

1. 区域分类
	1. 传输区域：除了本区域发起的流量和访问本区域的流量外，还需要承载源或目的IP不属于此区域的流量
	2. 末端区域：只承载本区域发起的流量和访问本区域的流量
2. 特殊区域
	1. OSPF特殊区域只能是末端区域，并且**末端区域不可加载vlink**。



## stub区域

### 通用硬性约束（Stub / Totally Stub 全部遵守）

1. 区域内**不允许 ASBR**：不能重分布静态、直连、其他协议路由，不会产生 Type-5 外部 LSA；
2. 不能作为虚链路中转区域；
3. 区域所有路由器（ABR + 内部路由器）必须统一配置 stub，否则邻居无法建立；
4. 仅允许 LSA-1、LSA-2 在区域内泛洪；
5. ABR 会向区域下发一条默认路由 0.0.0.0/0，区域路由器访问外部、其他区域全部走默认；
6. 不能是 Area 0 骨干区域（骨干不能配置 stub）。

### 标准stub区域
1. 允许进入本区域的 LSA
	1. Type-1 路由器 LSA：区域内设备自身产生，区域内泛洪
	2. Type-2 网络 LSA：区域内广播 / NBMA 网段产生，区域内泛洪
	3. Type-3 区域间汇总 LSA：**其他普通区域的明细网段路由全部下发**
	4. Type-3 默认路由 LSA：ABR 自动生成 0.0.0.0/0 下发到 Stub 区域
2. 禁止进入本区域的 LSA
	1. Type-4 ASBR 汇总 LSA：不存在，因为无外部路由
	2. Type-5 AS 外部 LSA：完全过滤，不进区域
	3. Type-7 NSSA 外部 LSA：Stub 不支持 7 类

### 完全stub区域
## 1. LSA 泛洪规则（和 Stub 核心区别）

1. 允许进入本区域的 LSA

	1. Type-1、Type-2：区域内本地 LSA，正常泛洪
	2. **仅一条特殊 Type-3 LSA：默认路由 0.0.0.0/0**
2. 严格禁止进入的 LSA
	1. 所有普通 Type-3 区域间明细 LSA（其他区域网段全部屏蔽，不下发明细）
	2. Type-4、Type-5 全部外部相关 LSA 过滤

| 对比项            | Stub 末梢区域            | Totally Stub 完全末梢                  |
| -------------- | -------------------- | ---------------------------------- |
| 配置关键字          | 所有路由器：`stub`         | ABR：`stub no-summary`；内部路由器：`stub` |
| 允许 LSA-1/2     | 允许                   | 允许                                 |
| 其他区域明细 3 类 LSA | 下发全部明细               | 完全屏蔽，一条都不下发                        |
| 默认 3 类 LSA     | 存在，用于访问外部路由          | 存在，所有跨区域 + 外网都靠这条默认                |
| LSA4/5         | 全部过滤                 | 全部过滤                               |
| 区域 ASBR        | 不允许存在                | 不允许存在                              |
| 能否配 vlink      | 不能                   | 不能                                 |
| Area 0 可用？     | 不可                   | 不可                                 |
| 区域路由内容         | 域内路由 + 其他区域明细 + 默认路由 | 仅域内路由 + 一条默认路由                     |
| 资源占用           | 中等                   | 最小                                 |
| 排障便利性          | 好，能看到跨区域网段           | 较差，跨区域路由全隐藏                        |
| 典型场景           | 中等规模分支，需访问多区域业务      | 小型门店、单网段末端分支                       |

## NSSA区域

NSSA区域不会生成默认路由，必须手动加nssa default-route-advertise才会生成默认路由
totally nssa区域才会自动生成默认路由
